# 0.1.9 — 2026-05-28

## Changed

- **`@cyanheads/mcp-ts-core`** `^0.9.9 → ^0.9.13` — framework adoption picks up several HTTP transport improvements:
  - **`MCP_HTTP_MAX_BODY_BYTES`** — configurable inbound request-body cap (default 1 MiB); oversized requests rejected with 413 before SDK body parsing ([mcp-ts-core #157](https://github.com/cyanheads/mcp-ts-core/issues/157))
  - **HTTP session-init gate** — stateful HTTP mode now rejects non-`initialize` requests that arrive without `Mcp-Session-Id` with HTTP 400, preventing uninitialized sessions ([mcp-ts-core #154](https://github.com/cyanheads/mcp-ts-core/issues/154))
  - **`httpErrorHandler` log-level split** — expected client errors (401, 403, 400, 404) downgraded from full `ErrorHandler.handleError` pipeline to `logger.warning`, reducing log noise ([mcp-ts-core #158](https://github.com/cyanheads/mcp-ts-core/issues/158))
  - **`GET /mcp` `server.keywords`** — `package.json` `keywords` now surfaced on the HTTP status response ([mcp-ts-core #165](https://github.com/cyanheads/mcp-ts-core/issues/165))
- **`@biomejs/biome`** `^2.4.15 → ^2.4.16`
