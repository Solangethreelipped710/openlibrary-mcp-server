# 0.1.10 — 2026-05-30

## Changed

- **`openlibrary_search_books`**, **`openlibrary_search_authors`**, **`openlibrary_get_subject`** — empty-result guidance and query context now surface in a typed `enrichment` block reaching both the `structuredContent` JSON and the `content[]` markdown trailer. The former `output.message` field is replaced by `enrichment.notice` (all three tools) and `enrichment.queryEcho` (`search_books` only — echoes active field filters when more than one is in use).
- **`openlibrary_search_books`**, **`openlibrary_search_authors`** — removed the unreachable `no_results` error contract (closes [#4](https://github.com/cyanheads/openlibrary-mcp-server/issues/4)). Empty results return a success response with an `enrichment.notice`; `ctx.fail('no_results', …)` was never called and `data.reason` was never populated.
- **`@cyanheads/mcp-ts-core`** `^0.9.13 → ^0.9.16`

## Fixed

- **`docs/design.md`** — removed stale `no_results` error entry from `openlibrary_search_books` errors section.
