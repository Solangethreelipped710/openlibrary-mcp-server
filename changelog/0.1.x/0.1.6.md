# 0.1.6 — 2026-05-24

## Fixed

- **`openlibrary_get_subject`**: removed `ebooks_only` input and the upstream `ebooks=1` query param — the Open Library subjects API silently ignores the param, so the filter had no effect ([#1](https://github.com/cyanheads/openlibrary-mcp-server/issues/1)).
- **`openlibrary_get_subject`**: handler now returns a `message` field when `work_count` is 0, explaining that subjects are case-sensitive and suggesting lowercase or alternate forms ([#2](https://github.com/cyanheads/openlibrary-mcp-server/issues/2)).
